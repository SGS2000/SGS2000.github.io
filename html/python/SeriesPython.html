Predicci√≥n de precios de Bitcoin¬∂

Se intenta predecir los precios de Bitcoin, utilizando datos de las cotizaciones hist√≥ricas entre 2015 y 2023. Se comparan dos t√©cnicas: un modelo ARIMA de series de tiempo y un bosque aleatorio.

1) Carga de datos¬∂

Los datos se descargan de Yahoo! Finance. Se obtiene la cotizaci√≥n diaria del BTC en d√≥lares entre el 01/01/2015 y el 31/12/2023. El archivo .csv es convertido a .xlsx e importado a Python, estableciendo la fecha como √≠ndice.

In¬†[7]:
#Predicciones
predicciones = list()

for t in range(len(test)):
 fecha_previa = train.index[-1]
 fecha_nueva = fecha_previa + pd.DateOffset(days=1)
 modelo_arima = ARIMA(train, order=(2,1,2))
 modelo_arima_fit = modelo_arima.fit()
 nueva_pred = modelo_arima_fit.forecast()[0]
 predicciones.append(nueva_pred)
 val_obs = test[t]
 train[fecha_nueva] = val_obs

predicciones_serie = pd.Series(predicciones, index=test.index)

Se grafican las predicciones junto a los verdaderos valores y se calcula tambi√©n el MAPE (error de porcentaje medio absoluto).

4) Random Forest¬∂

Se importan las librer√≠as necesarias para ajustar un modelo de bosques aleatorios (random forest). Se utilizar√° la cotizaci√≥n diaria como variable respuesta y la cotizaci√≥n del d√≠a anterior como variable expicativa. Una vez m√°s, las observaciones de noviembre y diciembre de 2023 son apartadas para ser predichas posteriormente por el modelo.

In¬†[9]:
#Liibrer√≠as
import warnings
from sklearn.ensemble import RandomForestRegressor
from sklearn.model_selection import GridSearchCV

#Calcular precios del d√≠a anterior
x = datos["BTC"].dropna().shift(1)
y = datos["BTC"].dropna()
precios = pd.concat([y,x], axis=1, keys=['y', 'x']).dropna()
        
#Dividir en conjunto de entrenamiento y prueba
train, test = precios[:-n_prueba], precios[-n_prueba:]

Se realiza el ajuste de los hiperpar√°metros. Para ello, se define una grilla con distintas opciones para cada hiperpar√°metro. Se aplica validaci√≥n cruzada para encontrar la combinaci√≥n √≥ptima. El modelo es finalmente ajustado con dicha combinaci√≥n de hiperpar√°metros.

In¬†[10]:
#Tuneo de hiperpar√°metros
grilla_parametros = {
    'n_estimators': [10, 50, 100, 250, 500, 1000],
    'min_samples_leaf': [1, 5, 20],
    'criterion': ['friedman_mse', 'squared_error', 'absolute_error']
}

busqueda_parametros = GridSearchCV(RandomForestRegressor(), param_grid=grilla_parametros)
    
busqueda_parametros_fit = busqueda_parametros.fit(
    train["x"].values.reshape(-1, 1),
    train["y"].values.reshape(-1, 1))
    
#Ajuste del modelo
modelo_rf = RandomForestRegressor(random_state = 30062000, 
                                  n_estimators = busqueda_parametros_fit.best_params_["n_estimators"],
                                  min_samples_leaf = busqueda_parametros_fit.best_params_["min_samples_leaf"],
                                  criterion = busqueda_parametros_fit.best_params_["criterion"])

modelo_rf_fit = modelo_rf.fit(train["x"].values.reshape(-1, 1), train["y"].values.reshape(-1, 1))

print("Par√°metros del modelo: ", modelo_rf_fit.get_params())
